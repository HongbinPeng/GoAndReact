## 学习说明

这个目录主要存放我在学习 Go 语言过程中跟着课程内容敲的练习代码。和课后作业相比，这里的代码更偏向知识点验证和小功能练习，目的是帮助我理解 Go 的基础语法、常用数据结构、函数、包管理、接口以及并发编程相关内容。

在练习过程中，我不是单纯照着示例复制代码，而是会在原有代码基础上加入自己的理解，比如增加打印输出、补充注释、修改参数、观察运行结果，借此更直观地理解每个知识点。

## 本周练习内容概览

### 1. 字符串与 rune

对应目录：

- `abautstring/`

我在这部分主要练习了：

- 字符串的基本输出方式。
- 多行字符串的写法。
- `rune` 的使用方式。
- 中文字符串的反转处理。

通过 `abautstring/rune/str.go` 中的代码，我实际体验了为什么 Go 中处理中文字符串时不能简单按字节来反转，而要先转换成 `[]rune` 再处理。这个练习让我对字符串、字符和编码之间的关系有了更直观的认识。

### 2. 数组与切片

对应目录：

- `sliceandarray/`

我在这部分主要练习了：

- 数组的定义、初始化和遍历。
- 切片的定义、初始化和 `make` 的使用。
- `len` 和 `cap` 的区别。
- `append` 的使用。
- 可变参数本质上也是切片。
- `copy` 的行为以及值类型和指针类型拷贝的区别。
- 子切片与原切片共享底层数组的问题。

在这部分练习中，我对 Go 里“数组是值类型、切片是引用底层数组的轻量结构”这一点印象很深。特别是在 `arrayandslice.go` 和 `slice.go` 中，通过修改拷贝后的数组和切片，能清楚看到两者行为上的差异。

### 3. map 的基本操作

我在这部分练习了：

- `map` 的创建。
- 键值对的增删改查。

这部分内容虽然不长，但让我理解了 `map` 是 Go 中非常常用的一种数据结构，也熟悉了 `value, ok := map[key]` 这种常见写法。

### 6. 接口相关练习

对应目录：

我在这部分练习了：

通过 `Dog`、`Robot`、`Speaker`、`Pet` 这些示例，我逐渐理解了 Go 接口更强调“行为约束”，而不是传统面向对象语言里的继承关系。这部分内容对我后面理解解耦和抽象帮助比较大。

### 7. goroutine、channel 与并发同步

对应目录：

- `goroutine/`

我在这部分主要练习了：

- 使用 `go` 关键字启动 goroutine。
- 使用 channel 在 goroutine 之间传递数据。
- 利用 channel 做同步控制。
- 简单的生产者消费者模型。
- 多生产者、多消费者模型。
- `select` 的基本使用。
- `sync.Mutex`、`sync.RWMutex`、`sync.WaitGroup`、`sync.Once`。

- 死锁场景的理解。

这一部分是我觉得最有挑战、也最有收获的内容。前面的基础语法大多是顺序执行，而并发部分开始涉及多个 goroutine 同时运行、资源竞争、同步等待等问题。通过这些练习，我对 Go 的并发模型有了初步认识，也知道了为什么并发程序必须重视同步和共享数据安全。

## 学习过程中遇到的问题

在练习这些代码时，我也遇到了一些比较典型的问题：

1. 切片和数组容易混淆。  
   一开始我会把切片当成“动态数组”简单理解，但在实际修改元素、观察地址和拷贝结果之后，才真正理解它和数组的区别。

2. 字符串和中文处理不够直观。  
   如果直接按字节处理中文字符串，很容易出现结果错误。通过 `[]rune` 的练习，我才逐渐明白为什么需要区分字节和字符。

3. 值接收者和指针接收者容易弄混。  
   只有在自己实际打印结构体字段变化之后，才真正理解方法到底有没有修改到原对象。

4. 并发部分理解难度明显更高。  
   特别是 goroutine、channel、锁、死锁这些内容，单看概念容易懂，但真正自己写的时候很容易出错，所以我只能边写边试、边输出边观察。
